# 函数内 static、类 static：为什么全程序只一份 

核心原因：**内存布局 + 链接属性 + 初始化规则**

先记结论：
**函数内static、类static 都存放在全局静态区，不在栈、不在对象里，所以全程序永远只有一份。**

## 一、先搞懂程序三大内存区
1. **栈区**：普通局部变量，函数调用就创建，结束就销毁，每次都是新的。
2. **堆区**：new/malloc 出来的，自己管理。
3. **静态全局区（数据段/BSS段）**：
   放**全局变量、函数内static、类static**
   
   > 程序一启动就分配，**整个程序生命周期只分配一次内存**，天然就只有一份。

---

## 二、逐个解释为什么「只有一份」

### 1. 函数内部 static 变量
```cpp
void foo() {
    static int x = 10;
    x++;
}
```
#### 为什么只有一份？
1. **不存栈上，存静态全局区**
普通局部变量在栈，每次进函数新建；
`static int x` 直接放进**静态数据段**，**整个程序就一块固定内存**。

2. **初始化只执行一次**
编译器做了隐形标记：加一个隐藏布尔标志，
第一次调用函数才初始化，后面再调用**跳过初始化，复用原来那份值**。

3. 不管调用多少次、多线程调用，**都共用同一个内存地址**。

---

### 2. 类的 static 成员
```cpp
class A {
public:
    static int cnt;
};
int A::cnt = 0;
```
#### 为什么只有一份？
1. **不属于任何对象，属于类本身**
普通成员变量：每个对象单独一份，各自有自己的内存。
`static` 成员**不占对象内存**，直接存到**全局静态区**。

2. **全局链接、全工程唯一**
类静态成员默认**外部链接**，整个项目只能定义一次，所有对象、所有文件共享**同一块内存**。

3. 不管 new 多少个对象、栈上创建多少对象，**都共用同一个 static 成员**。

---

## 三、对比：文件顶层 static 为什么不是一份？
```cpp
// a.cpp
static int n;
// b.cpp
static int n;
```
虽然也在静态区，但它是**内部链接**：
链接器给**每个 .cpp 单独分配一份独立静态内存**，互相隔离、互不共享，所以每个文件各有一份。

---

## 四、一句话本质总结
- **函数内static / 类static**：
  放进**全局静态区 + 外部链接/统一符号** → **全程序唯一一份**
- **文件顶层static**：
  放进**静态区但内部链接** → **每个文件独立一份**

---

## 五、极简记忆
**只要不在栈、不属于对象，扔到全局静态区的，天然就只有一份；
被 static 隔离成文件私有，就一人一份。**

# 内部链接（Internal Linkage）大白话讲透
一句话核心：
**被标记成「内部链接」的变量/函数，只给自己当前 `.cpp` 文件用，链接器直接把它锁在本文件里，别的文件完全找不到、不能共用、不会冲突。**

## 1. 先对比外部链接
- **外部链接**：
  全局符号，整个工程所有 `.cpp` 共享**同一个变量/函数**，跨文件能 `extern` 引用。
- **内部链接**：
  符号**仅限本文件私有**，别的文件看不见、连不上，每个文件自己玩自己的。

## 2. 谁自带内部链接
就三类，记死：
1. **文件顶层加 `static`**
```cpp
// 本文件专属，别的文件访问不到
static int a;
static void func(){}
```
2. **命名空间里写 `static`**
```cpp
namespace NS {
    static int x; // 只当前文件的NS里能用，别的文件看不到
}
```
3. **匿名命名空间里所有东西**
```cpp
namespace {
    int n; // 等价于 static，本文件私有，内部链接
}
```

## 3. 最关键特点（必懂）
### 特点1：各文件副本互相独立

```cpp
// a.cpp
static int val = 10;

// b.cpp
static int val = 20;
```
两个 `val` **不是同一个**，各有各的内存、互不干扰，不会报重定义错误。

### 特点2：不能跨文件 `extern` 引用
你在别的文件写：
```cpp
extern int val;
```
链接器找不到，直接**链接报错**。

### 特点3：作用域 + 链接双重限制
- 作用域：本文件内可用
- 链接属性：不对外暴露符号，隔离在当前编译单元

## 4. 和你之前的疑问串起来
1. **文件顶层 static = 内部链接** → 每个文件单独一份
2. **函数内 static、类 static = 外部链接** → 全程序只一份

## 5. 极简口诀
**内部链接 = 本文件私有化**
藏起来不给别的文件看，自己文件随便用，同名也不打架。

# 链接器（Linker）超通俗精讲
## 一、先一句话定性
**编译器**：把每个 `.cpp` 单独翻译成 `.o/.obj` 目标文件
**链接器**：把所有 **.o 目标文件 + 库文件** 拼在一起，合并成最终可执行文件（`.exe`/程序）

---
## 二、编译 → 链接 完整流程
1. 你写很多个 `a.cpp`、`b.cpp`、`main.cpp`
2. **编译器**：逐个单独编译，生成 `a.o`、`b.o`、`main.o`
   每个 `.o` 都是**半成品**，只懂自己内部的代码，不知道别的文件
3. **链接器登场**：
   把所有 `.o` 捆起来、修好跨文件的函数/变量调用、合并全局符号、分配最终内存地址，生成 **exe**

---
## 三、链接器核心干3件事
### 1. 符号解析
每个变量、函数都叫**符号**
链接器负责：
- 找到 `extern int x` 到底定义在哪个 `.o`
- 找到调用的函数 `func()` 真身在哪

### 2. 地址重定位
编译阶段只是**临时地址**
链接器给所有全局/静态变量、函数分配**最终真实内存地址**，修好所有跳转、调用地址。

> **为什么要修好**：编译单个 .cpp 时，编译器根本不知道函数 / 变量最终会放在内存哪个真实位置，只能先填「临时空位」，最后链接器必须把这些空位改成真实地址，不然程序一跑就崩。

### 3. 合并与去重、规则管控
按**链接属性**处理符号：
- **外部链接**：全工程只能有**一个定义**，所有文件共用这一份

- **内部链接**：每个 `.o` 自己留一份，互不干扰，允许同名

  > **外部链接**：放**公共大仓库**，所有人都能开锁进来共用同一份
  >
  > **内部链接**：也在同一个大仓库，但**单独锁了一个小隔间**，**只有当前这一个文件能解锁用**，别的文件打不开、进不来

---
## 四、用生活比喻超好懂
- 每个 `.cpp` = 单独一章书稿
- 编译器 = 单独写完每一章
- 链接器 = 总编：
  把所有章节装订成书、统一目录、修好交叉引用、删掉重复内容、定稿成完整书籍

---
## 五、和你刚才学的「内部/外部链接」直接挂钩
1. **外部链接**
告诉链接器：这个符号**公开共享**，所有 `.o` 共用同一个，不许重复定义。
2. **内部链接**
告诉链接器：这个符号**只给自己这一章用**，别的章节别碰，各章可以同名、互不影响。

---
## 六、常见报错都是链接器报的
- `重复定义 multiple definition`：外部链接符号在多个 `.cpp` 都定义了
- `未定义引用 undefined reference`：声明了 `extern`，但整个工程没找到实体定义

---
## 七、极简总结
**编译器：单文件翻译**
**链接器：多文件合并、修地址、管符号共享规则**
内部/外部链接，就是**给链接器下达的符号隔离/共享命令**。
